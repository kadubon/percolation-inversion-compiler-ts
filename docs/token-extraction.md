# Token Extraction

The ALT token path is trace to segmentation, candidate mining, canonicalization, ablation design, leakage check, dependency graph, minimal interface, verifier binding, and packet proposal. Extraction never executes the token and never silently infers mechanism validity.

A useful intervention is not automatically an abstraction token. Token admissibility requires provenance, reusable handle, counterfactual contrast, guard/failure contract, mechanism-mediated reuse, leakage exclusion, dependency closure, verifier binding, cost/risk accounting, transport scope, lifecycle, and authority/capability envelope.

Boundary markers for agents and CCR audits: Token admissibility is a candidate check, not capital admission. Accepted tokens remain unsettled until external obligations, transport scope, lifecycle costs, and authority/capability envelopes are checked.
